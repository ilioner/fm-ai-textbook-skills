# 路由指引（routing-guide）

> 本文是 `navigator` 技能的参考资料：意图 → 子技能的路由锚点，以及编排代跑的注意事项。
> 子技能清单由 `discover_skills.py` 运行时动态得到，本文的锚点仅辅助意图判断，
> **不作为写死的技能列表**；实际以发现结果为准。

## 一、意图 → 子技能 路由锚点

用户用自然语言表达需求时，可参考下表把意图映射到子技能。表中关键词是**辅助线索**，
最终仍以发现到的子技能 `description` 为准（新增技能时本表可能未收录，按 description 判断）。

| 用户意图 / 关键词 | 目标子技能 | 产出 |
|---|---|---|
| 知识图谱、图谱、节点、关系、可视化、预览页、脑图 | `knowledge-graph-extract` | `output/knowledgegraph/{lessonId}.json` + 预览页 |
| 知识点、KP、考点、教学目标、怎么教、怎么考、出题方向 | `guidance-learning` | `output/KP/{lessonId}.md` |
| 卡片、卡片数据、cardData、精华摘要、扁平化、卡片页面 | `extract-card-data` | `output/cardData/{lessonId}.json` |
| 课程思政、思政、育人、价值引领、思政方案、课堂活动 | `recommend-ideo-content` | `output/recommendIdeoContent/{lessonId}.md` |

判断原则：
- **显式指定优先**：用户直接点名技能 / 目录名 / 产出类型，直接选中，不再靠关键词猜。
- **一个意图命中多个**：如「把这批教材都处理一遍」——列出全部候选，让用户勾选要跑哪些、按什么顺序。
- **命中不明确**：宁可列菜单让用户选，不要凭模糊线索擅自代选。

## 二、编排代跑注意事项

导航技能代跑子技能时，**永远以该子技能的 `SKILL.md` 全文为准**，本文不复制其规则。
以下仅是通用注意事项：

1. **先读 SKILL 全文再动手**：代跑前完整读一遍目标子技能的 `SKILL.md`，
   把它的铁律、门禁、执行步骤、校验脚本都纳入执行，不凭记忆或本文简述代跑。
2. **门禁必须执行**：部分子技能有开跑前门禁（如 `knowledge-graph-extract` 的「最大层级」确认）。
   代跑时该问用户的仍要问，不能跳过。
3. **调子技能自己的校验脚本**：产出后运行该子技能的校验器（如 `validate.py` / `validate_kp.py`），
   通过后才算这一步完成。
4. **产出目录各归各**：每个子技能有固定的 `output/<子目录>/`，不要串目录。
5. **默认不覆盖**：多数子技能默认跳过已存在的产出，仅在用户明确要求重做时覆盖——遵循子技能自身约定。
6. **目标目录一次确认、全程复用**：批量跑多个子技能时，目标目录（含 `markdown/`）确认一次即可，
   后续子技能复用同一目录。

## 三、链式衔接话术参考

每个子技能跑完后，向用户报告产出并引导下一步。参考：

> 已完成【<子技能名>】，产出在 `output/<子目录>/`（共 N 个 lesson）。
> 还需要对同一批教材跑其它技能吗？可选：<剩余子技能清单>。
> 回复要跑的技能（可多选、可指定顺序），或回复「结束」。

- 若用户一开始就多选了若干技能，按既定顺序跑，跑完一个自动进入下一个，无需每个都重新确认，
  但每个子技能自身的门禁（如层级确认）仍要执行。
- 全部跑完或用户回复结束时，做一次总汇报：列出本轮产出的所有 `output/` 子目录。
