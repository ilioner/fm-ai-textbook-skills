# 课程思政实施方案生成

以职业教育教学设计者视角，逐篇理解教材内容，将价值引领自然嵌入具体知识点、核心概念或实操步骤。

## 铁律

1. 逐个 lesson 完整阅读，不跨 lesson 混用知识点。
2. 教材事实必须忠于原文。术语、定义、步骤、条件、数值和因果关系不得篡改或臆造。
3. 每个主题至少绑定一个知识性原文块。引文必须从解析结果机械复制，并保留对应 `data-hash`。
4. 思政设计必须服务于教材知识学习，不脱离知识点另行说教。
5. 每课提炼 3-5 个主题。外部事实仅能写成不含具体虚构细节的“可选案例方向”。
6. 输出使用标题、段落和列表，禁止 Markdown 表格和 HTML 表格。

## 执行流程

### 1. 解析教材

```bash
python3 <技能目录>/scripts/parse_lesson.py \
  --input-dir <目标目录>/markdown \
  --output-dir <临时解析目录>
```

先确认文件名三段 ID 与 frontmatter 一致。解析结果中的 `text` 和 `dataHash` 是不可篡改数据源。

### 2. 逐课理解与选点

完整阅读单课原文，梳理核心概念、分类、原理、流程、规范和实操步骤。优先选择能自然承载职业道德、安全责任、质量意识、协作精神、创新意识、法治意识或社会责任的知识点，但主题名称必须由本课内容决定。

不要把任务目标、素质目标或导入语直接改写成方案。它们可辅助判断方向，每个主题仍须锚定知识正文或操作步骤。

### 3. 生成方案

输出到 `<目标目录>/output/recommendIdeoContent/{lessonId}.md`。严格遵守
`references/output-format.md`：

- 写明 lesson 元信息和总体设计思路；
- 生成 3-5 个连续编号主题；
- 每个主题包含全部固定栏目；
- “教材原文依据”只放逐字原文和对应定位；
- 教学活动写清教师组织、学生任务、课堂产出与评价方式；
- 育人目标使用可观察行为描述。
- 每个主题必须设计独立的岗位情境、学生任务、课堂产物、评价标准、问题和行为证据。
  不得只替换知识点名称后复用同一套“知识判断—风险分析—岗位行动”等通用话术。

### 4. 忠实性回查

逐主题回到该 lesson 核对：

- 切入点是否确属本课知识；
- 原文引文是否完整复制；
- 分析是否改变教材事实；
- 活动是否能同时推进知识、能力与价值目标；
- 情境、产物和评价是否只有本知识点成立，换成其他课题后是否会失真；
- 问题是否包含本知识点特有的判断冲突，目标是否能由课堂中的具体行为或产物观察；
- 是否出现未经教材支持的具体外部事实。

### 5. 强制校验

```bash
python3 <技能目录>/scripts/validate_output.py \
  --input-dir <目标目录>/markdown \
  --output-dir <目标目录>/output/recommendIdeoContent
```

校验失败时修正后重跑，直至失败数为 0。

## 组件

- `scripts/parse_lesson.py`：解析文件名、frontmatter、标题与带 hash 原文块。
- `scripts/validate_output.py`：校验输出集合、结构、表格、hash 归属和引文逐字一致。
- `references/output-format.md`：非表格 Markdown 输出契约。

本技能适用于同一导出格式的任意职业教育教材，不依赖当前测试教材的具体学科。
