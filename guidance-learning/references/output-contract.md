# KP 输出格式契约

> 本文件定义 `output/KP/{lessonId}.md` 的严格格式。示例仅用于演示**格式**，
> 不代表本技能只处理该科目——技能适用任意同系统教材。

## 文件级规则

- 文件名：`{lessonId}.md`，`lessonId` 取自输入文件名第三段（并与 frontmatter 一致）。
- 文件内容**只含** KP 清单，**禁止**：
  - 前言 / 介绍 / 说明文字
  - 总结 / 结语
  - 代码围栏（```）
  - 除 KP 块之外的任何段落
- 每个输入 lesson 对应且仅对应一个输出文件（一一对应）。

## KP 块规则

- KP 从 `KP1` 开始**连续编号**（KP1、KP2、KP3…），不得跳号、不得重复。
- 每个 KP 块**且仅**含四部分，**顺序固定**：

```
## KPn: <简短标题>
* 教学目标: <一句话，学习者掌握后能理解什么/做什么>
* 核心概念: <最精炼的本质与关键>
* 提问方向: <设计一道检验该 KP 的选择题的指令>
```

- 标题行：`## KPn: ` 前缀 + 标题（`n` 为连续序号）。
- 三个字段行：以 `* ` 开头，字段名后紧跟 `: `，顺序必须是 教学目标 → 核心概念 → 提问方向。
- 字段不得缺失、不得重复、不得乱序、不得新增其它字段。

## 字段写法

| 字段 | 要求 |
|---|---|
| KP标题 | 简短精炼，概括知识点核心；名词/短语式 |
| 教学目标 | 说明学习者掌握后应能【理解什么】或【做什么】；知识范围受原文约束 |
| 核心概念 | 用最精炼的词/短句点出本质与关键 |
| 提问方向 | 设计选择题的**指令**——只说考什么、怎么考；**不写**题干/选项/答案 |

## 正确示例（格式演示）

```
## KP1: 大气三大基本要素的识别
* 教学目标: 准确识别并理解气温、气压和空气湿度作为大气三大基本气象要素的核心地位。
* 核心概念: 三大要素定义气象状态。
* 提问方向: 设计一个选择题，要求用户从多个选项中选出哪三个是被称为"三大气象要素"的物理量。

## KP2: 气温的本质与影响
* 教学目标: 理解气温是空气分子平均动能的宏观表现，并掌握其随高度在对流层内降低的基本规律。
* 核心概念: 气温反映分子动能，对流层内随高度递减。
* 提问方向: 设计一个选择题，考察用户对"气温"物理本质的理解及其在对流层中的垂直变化趋势。

## KP3: 平流层与对流层的特点对比
* 教学目标: 掌握平流层气流平稳、能见度高的特点，并能与对流层对流强烈、天气多变的特点相区分。
* 核心概念: 平流层气流平稳利于飞行，与对流层形成对比。
* 提问方向: 设计一个选择题，将平流层与对流层的核心特征并列，考察用户的区分能力。
```

## 反例（会被 validate_kp.py 拒绝）

- ❌ 文件开头有「以下是本课的核心知识点：」等前言
- ❌ 用了 ```markdown 代码围栏包裹
- ❌ 编号跳号（KP1、KP3）或重复（两个 KP2）
- ❌ 缺少某个字段（如只有教学目标、核心概念，没有提问方向）
- ❌ 字段顺序错乱（提问方向写在核心概念前面）
- ❌ 提问方向里直接写出了题干和 ABCD 选项
- ❌ KP 块之间插入了额外说明段落
