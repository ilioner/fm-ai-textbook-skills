# 核心知识点抽取（guidance-learning）

以「AI 教学设计师」的视角，把教材 markdown 提炼成结构化的**核心知识点（KP）清单**。
每个 KP 从「如何教、如何考」出发，具备明确的教学与考查价值。

## 何时使用

用户在某个「目标目录」下需要抽取核心知识点，且该目录含 `markdown/` 文件夹，
文件命名为 `textbookId_unitId_lessonId.md`。

## 铁律（最高优先级，任何时候不可违背）

1. **忠于原文知识**：KP 涉及的术语、定义、事实、分类、步骤、条件、数值、因果方向，
   必须与教材原文一致，**不得改写为不同含义、不得篡改**。
2. **不臆造**：教材未明确给出的知识，**不得补充、不得推断为事实**。宁可少提，不可编造。
3. **只提炼、不出题**：「提问方向」只给出**设计选择题的指令**，绝不生成题干、选项或答案。
4. **结构由脚本硬校验**：写完必须用 `validate_kp.py` 校验格式，不合规就修正重来。
5. **默认不覆盖**：已存在的 `output/KP/{lessonId}.md` 默认不动，仅在用户明确要求重做时覆盖。

## 执行流程

### 1. 清点输入（脚本）
```bash
python3 <技能目录>/scripts/prepare_batch.py <目标目录>
```
输出批处理清单（每个 lesson 的三 ID、输出路径、是否已存在）。若某文件名或 frontmatter
三 ID 不一致，清单会标 `errors`——先解决再抽取。已存在输出的 lesson 默认跳过（除非用户要求覆盖）。

### 2. 逐篇完整阅读（你来做）
对每个 lesson 的 md 文件，**完整读一遍原文**，理解教学语境：讲了哪些知识、彼此什么关系、
哪些是需要辨析的独立概念、哪些是可考的要点。不要只扫标题就动手。

### 3. 提炼 KP（你来判断）
识别**可教、可考、且原文有依据**的核心知识点，逐个提炼。

**KP 粒度原则：**
- 每个 KP 有**独立且明确**的教学/考查价值。
- **不要把同一知识机械切碎**成多个琐碎 KP。
- **不要把需要辨析的独立概念合并**成一个（如「对流层」和「平流层」的对比要点，各自成立时应分开）。
- KP 数量随教材内容多寡而定，没有固定值。

**剔除非知识内容**：教学脚手架（教学目标/任务目标/素质目标/能力目标、任务描述、
图片说明、导入语、以及练习题/检验题本身）不单独成 KP。但练习题**指示了考点方向**，
可据此判断哪些知识点值得作为 KP、以及其提问方向。

### 4. 写出 KP 清单（严格格式）
写到目标目录 `output/KP/{lessonId}.md`（目录不存在则创建）。
详细格式与正确示例见 `references/output-contract.md`。**核心格式：**

- 文件**只含** KP 清单，**禁止**任何前言、说明、总结、代码围栏（```）。
- KP 从 `KP1` 开始**连续编号**，不跳号。
- 每个 KP **且仅** 含以下四部分，顺序固定：
  ```
  ## KPn: <简短标题>
  * 教学目标: <学习者掌握后能理解什么/做什么>
  * 核心概念: <最精炼的本质与关键>
  * 提问方向: <设计一道检验该 KP 的选择题的指令>
  ```

**四字段写法：**
- **KP标题**：简短精炼，概括该知识点核心。
- **教学目标**：说明学习者掌握后应能【理解什么】或【做什么】。知识范围受原文约束。
- **核心概念**：用最精炼的词/短句点出本质与关键。
- **提问方向**：一个明确的**指令**，用于设计检验该 KP 的选择题——只说「考什么、怎么考」，
  **不写**题干、选项、答案。

### 5. 忠实性回查（写完必做）
逐个 KP 回到原文核对：
- 每个 KP 的知识点，原文是否**确有依据**？（无依据的删掉）
- 术语、数值、分类、因果方向是否与原文**一致**？（不一致的改回原文口径）
- 有没有把原文没说的内容**当成事实**写进去？（有则删）

### 6. 结构校验（脚本，强制）
```bash
# 校验整个 output/KP/ 与输入一一对应 + 每个文件的四字段结构
python3 <技能目录>/scripts/validate_kp.py <目标目录>
# 或只校验单个文件的结构
python3 <技能目录>/scripts/validate_kp.py --file output/KP/<lessonId>.md
```
必须「结构合规」。若报编号不连续/字段缺失/字段乱序/多余文本/输出集合不匹配，修正后重跑。

> 注意：脚本只能校验**结构与命名**。KP 的**知识忠实性**脚本无法证明——那由第 5 步的回查保证。

## 产出结构

```
目标目录/output/KP/
  {lessonId}.md     # 每个 lesson 一个 KP 清单
```

## 组件

- `scripts/prepare_batch.py` — 确定性输入清点：扫描 markdown/、解析文件名、校验 frontmatter 三 ID、报告输出冲突。
- `scripts/validate_kp.py` — 确定性产物校验：输出集合一一对应 + KP 四字段严格结构。
- `references/output-contract.md` — 输出格式契约 + 正确示例。

## 说明

- 本技能适用**任意科目**的同系统教材，SKILL 指令层不写死具体科目知识。
- KP 是纯语义提炼，无 data-hash 类可机械核对的锚点，故忠实性靠「逐篇完整读 + 写后回查」保证，
  脚本负责结构兜底。
